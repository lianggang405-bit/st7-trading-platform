# 调控机器人工作原理与使用说明

## 一、核心概念

### 1.1 什么是调控机器人？

调控机器人是平台后台的一个核心功能模块，允许管理员通过设置浮点值（float value）来调控特定交易对的行情价格，从而直接影响客户的交易盈亏。

### 1.2 工作原理

```
真实市场价格 (Binance等)
       ↓
   + 浮点值 (管理员设置)
       ↓
调控后的市场价格 (展示给客户)
       ↓
   影响客户盈亏
```

**核心公式：**

```
调控后价格 = 真实市场价格 + 浮点值
```

或者：

```
调控后价格 = 真实市场价格 × (1 + 浮点值%)
```

## 二、对客户盈亏的影响机制

### 2.1 做多（看涨）的情况

**场景：** 客户在价格 100 时做多 BTCUSD

| 真实价格 | 浮点值 | 调控后价格 | 客户盈亏 | 说明 |
|---------|--------|-----------|---------|------|
| 100 | +10 | 110 | +10 (盈利) | 价格上涨，做多盈利 |
| 100 | -10 | 90 | -10 (亏损) | 价格下跌，做多亏损 |

### 2.2 做空（看跌）的情况

**场景：** 客户在价格 100 时做空 BTCUSD

| 真实价格 | 浮点值 | 调控后价格 | 客户盈亏 | 说明 |
|---------|--------|-----------|---------|------|
| 100 | +10 | 110 | -10 (亏损) | 价格上涨，做空亏损 |
| 100 | -10 | 90 | +10 (盈利) | 价格下跌，做空盈利 |

### 2.3 盈亏计算公式

**做多（Long）盈亏：**

```
盈亏 = (平仓价格 - 开仓价格) × 合约数量 × 合约乘数
```

**做空（Short）盈亏：**

```
盈亏 = (开仓价格 - 平仓价格) × 合约数量 × 合约乘数
```

**关键点：** 平仓价格是经过调控后的价格，这直接决定了客户的盈亏。

## 三、调控机器人功能详解

### 3.1 核心功能

1. **新建调控机器人** - 为特定交易对创建调控机器人
2. **查看机器人详情** - 查看机器人的配置和历史
3. **编辑机器人参数** - 修改机器人的名称和浮点值
4. **删除调控机器人** - 移除不需要的调控机器人
5. **批量调控** - 同时管理多个交易对的调控

### 3.2 浮点值设置策略

#### 策略1：固定浮点值（推荐用于测试）

```
浮点值 = 固定数值（如 +50、-100）
```

**用途：** 测试系统、模拟极端行情

#### 策略2：比例浮点值（推荐用于实际运营）

```
浮点值 = 真实价格 × 百分比（如 1%、-0.5%）
```

**用途：** 微调市场走势、控制风险

#### 策略3：动态浮点值（高级）

```
浮点值 = f(时间、交易量、真实价格波动)
```

**用途：** 模拟真实市场波动、精细化调控

### 3.3 调控方向说明

| 浮点值方向 | 做多用户 | 做空用户 | 适用场景 |
|-----------|---------|---------|---------|
| **正值 (+)** | 盈利增加/亏损减少 | 盈利减少/亏损增加 | 希望做多用户盈利 |
| **负值 (-)** | 盈利减少/亏损增加 | 盈利增加/亏损减少 | 希望做空用户盈利 |
| **零 (0)** | 无影响 | 无影响 | 关闭调控 |

## 四、使用指南

### 4.1 新建调控机器人

**步骤：**

1. 进入管理后台 → 资源 → 调控机器人
2. 点击「新建机器人」按钮
3. 填写表单：
   - **名称**：给机器人起一个识别性的名字
   - **交易对**：选择要调控的交易对（如 BTCUSD）
   - **浮点值**：设置调控的数值（正数或负数）
4. 点击「创建」按钮

**示例：**

```
名称：BTCUSD 调控机器人（正向）
交易对：BTCUSD
浮点值：+50
```

这意味着：
- 当真实价格是 95000 时
- 客户看到的价格是 95050
- 做多的客户会多盈利 50 点

### 4.2 编辑调控机器人

**步骤：**

1. 在机器人列表中找到要编辑的机器人
2. 点击「编辑」按钮
3. 修改名称或浮点值
4. 点击「保存」按钮

**注意：** 交易对创建后**不可修改**，如需更换交易对请创建新机器人。

### 4.3 查看机器人详情

**步骤：**

1. 在机器人列表中找到要查看的机器人
2. 点击「查看」按钮
3. 查看完整信息：
   - 基本信息（ID、名称、交易对）
   - 当前浮点值
   - 创建和更新时间

### 4.4 删除调控机器人

**步骤：**

1. 在机器人列表中找到要删除的机器人
2. 点击「删除」按钮（垃圾桶图标）
3. 确认删除操作
4. 机器人被移除，该交易对恢复真实价格

**警告：** 删除操作不可恢复，请谨慎操作！

## 五、实际应用场景

### 5.1 场景1：新用户体验优化

**目标：** 让新用户第一次交易就盈利，提升用户体验

**配置：**
- 选择热门交易对（如 BTCUSD）
- 设置适中的正向浮点值（如 +20 到 +50）
- 监控用户反馈

**效果：** 新用户做多大概率会盈利，增强信心

### 5.2 场景2：风险控制

**目标：** 在市场剧烈波动时控制平台风险

**配置：**
- 对高风险交易对设置逆向浮点值
- 动态调整浮点值大小
- 配合其他风控措施

**效果：** 减少极端行情下的亏损

### 5.3 场景3：活动营销

**目标：** 特定时间段内促进交易活跃

**配置：**
- 对多个热门交易对设置正向浮点值
- 设置开始和结束时间
- 配合前端活动宣传

**效果：** 用户盈利增加，交易活跃

## 六、注意事项和最佳实践

### 6.1 注意事项

⚠️ **重要提醒：**

1. **合规风险**：请确保在法律允许的范围内使用此功能
2. **透明度**：建议向用户说明价格可能包含调控成分
3. **道德考量**：过度调控可能影响平台信誉
4. **数据备份**：定期备份机器人配置和调控历史
5. **风控配套**：应与其他风控措施配合使用

### 6.2 最佳实践

✅ **推荐做法：**

1. **小步快跑**：先小规模测试，确认效果后再扩大
2. **数据监控**：建立完善的监控体系，跟踪调控效果
3. **用户反馈**：收集用户反馈，及时调整策略
4. **梯度设置**：根据用户等级、交易量等设置不同梯度
5. **定期评估**：定期评估调控效果，优化参数设置

### 6.3 避免的误区

❌ **不要这样做：**

1. **过度调控**：浮点值过大可能导致价格异常
2. **频繁变动**：频繁修改浮点值会让用户困惑
3. **一刀切**：所有用户使用相同的调控值
4. **无风控**：仅依赖调控机器人，忽略其他风控
5. **不透明**：完全不向用户说明价格调控

## 七、技术实现细节

### 7.1 数据结构

**调控机器人表 (trading_bots)：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | integer | 主键ID |
| name | varchar | 机器人名称 |
| pair_id | integer | 关联的交易对ID |
| float_value | numeric | 浮点值 |
| created_at | timestamp | 创建时间 |
| updated_at | timestamp | 更新时间 |

### 7.2 API 接口

**获取机器人列表：**
```
GET /api/admin/trading/bots?page=1&limit=15&search=BTC
```

**创建机器人：**
```
POST /api/admin/trading/bots
{
  "name": "BTCUSD 调控机器人",
  "pairId": 23,
  "floatValue": 50
}
```

**更新机器人：**
```
PUT /api/admin/trading/bots/1
{
  "name": "BTCUSD 调控机器人（已更新）",
  "floatValue": 60
}
```

**删除机器人：**
```
DELETE /api/admin/trading/bots/1
```
