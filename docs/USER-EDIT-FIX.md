# 用户编辑功能测试

## 功能说明

用户编辑功能位于 `/admin/users/[id]` 页面，使用 `EditUserDialog` 组件实现。

## 组件位置

- **页面文件**: `src/app/admin/users/[id]/page.tsx`
- **编辑组件**: `src/components/admin/EditUserDialog.tsx`
- **API 接口**: `src/app/api/admin/users/[id]/route.ts`

## 支持的编辑字段

| 字段 | 类型 | 是否可编辑 | 说明 |
|------|------|----------|------|
| 邮箱 | email | ✅ 可编辑 | 必填字段 |
| 上级 | text | ❌ 只读 | 禁止修改 |
| 邀请码 | text | ✅ 可编辑 | 可选 |
| 用户类型 | text | ❌ 只读 | 禁止修改 |
| 用户等级 | text | ❌ 只读 | 禁止修改 |
| 状态 | select | ✅ 可编辑 | 正常/禁用/冻结 |
| 余额 | number | ✅ 可编辑 | 可调整用户余额 |
| 密码 | password | ✅ 可编辑 | 留空则不修改 |
| 备注 | text | ✅ 可编辑 | 可选 |
| 注册时间 | text | ❌ 只读 | 禁止修改 |
| 最后登录时间 | text | ❌ 只读 | 禁止修改 |

## API 接口

### PUT /api/admin/users/[id]

更新用户信息。

**请求体**:
```json
{
  "email": "user@example.com",
  "inviteCode": "abc123",
  "status": "正常",
  "balance": 1000.00,
  "password": "newPassword123",
  "remark": "用户备注信息"
}
```

**响应**:
```json
{
  "success": true,
  "user": {
    "id": 1,
    "email": "user@example.com",
    "invite_code": "abc123",
    "is_active": true,
    "balance": 1000.00,
    "remark": "用户备注信息",
    "updated_at": "2026-03-03T12:00:00Z"
  }
}
```

## 数据库更新

当用户点击"保存"按钮时，系统会：

1. 验证邮箱是否为空
2. 调用 PUT 接口更新数据库
3. 如果提供了密码，则使用 bcrypt 加密
4. 更新 `updated_at` 时间戳
5. 调用 `onSuccess` 回调刷新前端用户信息

## 前端刷新机制

用户编辑成功后，会触发以下刷新流程：

1. 关闭编辑对话框
2. 显示成功提示
3. 调用 `fetchUser()` 重新获取用户信息
4. 更新页面显示的所有用户字段

## 使用说明

1. 访问用户列表页面 `/admin/users/list`
2. 点击某个用户的"查看"按钮进入详情页
3. 点击右上角的"编辑"按钮打开编辑对话框
4. 修改需要编辑的字段
5. 点击"保存"按钮提交更改
6. 系统会自动更新数据库并刷新页面显示

## 注意事项

- 邮箱是必填字段，不能为空
- 密码字段留空则不修改原密码
- 余额只能是数字
- 状态只能选择"正常"、"禁用"或"冻结"
- 只读字段无法修改，包括：上级、用户类型、用户等级、注册时间、最后登录时间
