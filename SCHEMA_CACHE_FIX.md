# 🔧 Schema 缓存问题解决方案

## 问题说明

您已经成功创建了数据库表，但是应用仍然显示"数据库表不存在"。这是因为 **Supabase REST API 的 schema 缓存** 问题。

## ✅ 解决方案

### 方案1：等待自动刷新（最简单）

**Supabase 的 schema 缓存通常在 1-2 分钟内自动更新。**

请等待 1-2 分钟后，在应用设置页面点击 **"刷新状态"** 按钮。

### 方案2：手动刷新 Schema

如果您想立即解决这个问题，可以在 Supabase 中手动刷新：

1. 访问您的 Supabase 项目
2. 点击左侧菜单的 **Database**
3. 在页面顶部找到 **API** 标签
4. 点击 **Reset project URL** 或 **Reset password**（这会触发 schema 刷新）
5. 返回应用，点击 **"刷新状态"**

### 方案3：重启 Supabase 连接

1. 在 Supabase 中，点击左侧菜单的 **Settings**
2. 点击 **API**
3. 记下 `anon key`
4. 如果使用的是 Coze 环境，`coze_workload_identity` 会自动获取最新的 key
5. 返回应用，点击 **"刷新状态"**

## 📋 验证步骤

### 步骤1：在 Supabase 中确认表存在

在 Supabase SQL Editor 中执行：
```sql
SELECT table_name 
FROM information_schema.tables 
WHERE table_schema = 'public' 
  AND table_name IN ('trading_pairs', 'trading_bots');
```

应该返回：
```
table_name
-------------
trading_pairs
trading_bots
```

### 步骤2：在 Supabase 中确认数据存在

```sql
SELECT * FROM trading_pairs;
```

应该返回 4 条数据。

### 步骤3：在应用中刷新

1. 访问应用设置页面：`http://localhost:5000/admin/trading/setup`
2. 点击 **"刷新状态"** 按钮
3. 看到所有状态显示 ✅

## 🔍 为什么会发生这种情况？

### Schema 缓存机制

Supabase REST API 使用 schema 缓存来提高性能：
- 缓存包含：表结构、列信息、关系等
- 缓存更新：定期自动更新（通常1-2分钟）
- 手动触发：通过重置 API key 或项目 URL

### 为什么表存在但应用找不到？

1. **表已创建** - 在数据库中，表已经存在
2. **缓存未更新** - REST API 的缓存还是旧的
3. **返回错误** - API 根据缓存判断表不存在
4. **等待刷新** - 缓存更新后就能正常访问

---

## 💡 最佳实践

为了避免这个问题：

1. **提前创建表** - 在开发阶段就创建好所有需要的表
2. **使用迁移** - 使用 Supabase Migrations 管理表结构
3. **避免频繁修改** - 尽量避免在生产环境中频繁创建/删除表
4. **等待时间** - 创建表后等待1-2分钟再刷新应用

## ❓ 常见问题

### Q1: 我等了5分钟还是不行？

**A:** 请尝试：
1. 刷新浏览器页面（Ctrl + Shift + R）
2. 检查 Supabase 项目状态
3. 确认您在正确的项目中
4. 查看 Supabase 日志

### Q2: 如何确认缓存已更新？

**A:** 访问 Supabase 的 Database → API 页面，查看 table list 是否包含新表。

### Q3: 我可以禁用缓存吗？

**A:** 不可以。Schema 缓存是 Supabase 的核心功能，无法禁用。但这不是问题，因为缓存更新很快。
