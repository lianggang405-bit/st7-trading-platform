# 调控机器人数据库设置 - 常见问题解答 (FAQ)

## 📖 基础问题

### Q1: 为什么需要创建数据库表？

**A:** 调控机器人功能需要存储以下数据：
- 交易对信息（如 BTC/USDT、ETH/USDT）
- 调控机器人配置（是否启用、浮点值等）

如果不创建这些表，应用就无法存储和读取这些数据。

---

### Q2: 这个操作安全吗？

**A:** 完全安全！这个SQL脚本：
- ✅ 只创建必要的表结构
- ✅ 只插入示例数据（4个交易对）
- ✅ 不包含任何敏感信息
- ✅ 可以随时删除表（删除后不影响其他功能）

---

### Q3: 如果执行失败了怎么办？

**A:** 最常见的错误：

**错误1: "relation already exists"**
- 原因：表已经存在
- 解决：这是正常的，直接跳到第3步

**错误2: "permission denied"**
- 原因：没有足够的权限
- 解决：检查您是否登录了正确的 Supabase 账号

**错误3: "connection refused"**
- 原因：网络问题
- 解决：刷新页面重试

## 🔧 操作问题

### Q4: 我找不到 SQL Editor 在哪里？

**A:** SQL Editor 在 Supabase 仪表板的左侧菜单栏：
1. 登录 Supabase 后，看左侧边栏
2. 找到一个数据库图标（💾 或 📊）
3. 点击它，看到子菜单
4. 点击 "SQL Editor"

如果找不到：
- 点击左侧菜单的 "Database"
- 在页面顶部找 "SQL Editor" 标签

### Q5: 如何复制SQL代码？

**A:** 有三种方法：

**方法1: 从页面复制**
1. 在设置页面中，找到SQL代码框
2. 点击代码框右上角的 "复制SQL" 按钮
3. 代码会自动复制到剪贴板

**方法2: 手动选择复制**
1. 用鼠标选中所有SQL代码
2. 按 `Ctrl + C`（Mac: `Cmd + C`）
3. 粘贴到SQL Editor

**方法3: 从文件复制**
1. 打开项目目录中的 `init-trading-tables.sql` 文件
2. 全选并复制所有内容
3. 粘贴到SQL Editor

### Q6: 点击 Run 按钮后没有反应？

**A:** 可能的原因：

1. **正在执行中**：等待3-5秒，SQL执行需要时间
2. **SQL有语法错误**：检查代码是否完整复制
3. **浏览器问题**：刷新页面重试
4. **网络问题**：检查网络连接

## 🎯 验证问题

### Q7: 如何验证SQL执行成功了？

**A:** 看到以下任何一个都表示成功：

✅ SQL Editor 底部显示：**Success**
✅ 显示绿色对勾 ✓
✅ 显示查询结果：**Success 3 queries**

如果不确定，执行验证查询：
```sql
SELECT * FROM trading_pairs;
```

如果看到4条数据，说明成功了！

### Q8: 验证查询返回空结果？

**A:** 可能的原因：

1. **SQL执行失败**：重新执行创建表的SQL
2. **选错了数据库**：确保在正确的项目和数据库中执行
3. **表名不对**：确认表名是 `trading_pairs`（小写）

## 📱 应用问题

### Q9: 返回应用后仍然显示"需要创建数据库表"？

**A:** 可能的原因和解决方法：

1. **等待时间不够**
   - Supabase的schema缓存需要1-2分钟更新
   - 等待1-2分钟后刷新页面

2. **没有刷新状态**
   - 点击页面上的 "刷新状态" 按钮

3. **缓存问题**
   - 按 `Ctrl + Shift + R` 强制刷新浏览器

4. **表确实没有创建成功**
   - 回到 Supabase SQL Editor
   - 重新执行SQL
   - 查看是否有错误提示

### Q10: 点击"刷新状态"后一直显示加载中？

**A:**
1. 等待10-15秒，网络可能较慢
2. 如果超过30秒，刷新浏览器页面
3. 检查网络连接是否正常
4. 如果仍然卡住，关闭页面重新打开

## 🚨 紧急问题

### Q13: 执行SQL后，我的数据会丢失吗？

**A:** 不会！这个SQL：
- ✅ 只创建新表
- ✅ 不会删除任何现有数据
- ✅ 不会修改任何现有表

### Q14: 如果创建表失败了，会影响应用的其他功能吗？

**A:** 不会！
- 其他功能正常使用
- 只是调控机器人功能暂时不可用
- 可以随时重新创建表

### Q15: 我想删除这些表，可以吗？

**A:** 可以！在 Supabase SQL Editor 中执行：

```sql
DROP TABLE IF EXISTS trading_bots;
DROP TABLE IF EXISTS trading_pairs;
```

删除后，如果需要重新使用调控机器人，只需重新执行创建表的SQL即可。

## 💡 额外提示

### 设置成功后的使用：

1. **调控开关**：点击交易对旁边的开关，启用/禁用调控
2. **设置浮点值**：点击"调控"按钮，设置价格偏移量
3. **实时生效**：所有设置立即生效

### 浮点值说明：

- 浮点值 = 价格偏移量
- 实际价格 = 原始价格 + 浮点值
- 例如：原始价格 50000，浮点值 0.00123，则显示为 50000.00123
