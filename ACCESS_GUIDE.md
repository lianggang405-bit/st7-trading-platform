# 调控机器人页面访问指南

## ✅ 问题已解决

调控机器人页面现在可以正常访问了！

## 📋 访问方式

### 方式 1: 正确路径（推荐）
```
http://localhost:5000/admin/trading/bots
```
注意：路径最后是 `bots`（有 `s`）

### 方式 2: 旧路径（自动重定向）
```
http://localhost:5000/admin/trading/bot
```
注意：这个路径会自动重定向到上面的正确路径

## 🎯 如果显示404

如果您仍然看到404错误，请尝试以下步骤：

### 1. 清除浏览器缓存
- 按 `Ctrl + Shift + R`（Windows/Linux）
- 或 `Cmd + Shift + R`（Mac）

### 2. 确认访问路径
- 正确：`/admin/trading/bots`（有 `s`）
- 错误：`/admin/trading/bots/`（末尾有斜杠）

### 3. 检查服务状态
```bash
# 检查服务是否在运行
curl -I http://localhost:5000/admin/trading/bots
```

应该返回：
```
HTTP/1.1 200 OK
```

### 4. 查看日志
如果问题持续，检查日志：
```bash
tail -n 20 /app/work/logs/bypass/app.log
```

## 📊 API端点

页面使用的API：
- `GET /api/admin/trading/adjust` - 获取交易对列表
- `POST /api/admin/trading/adjust` - 创建/更新调控机器人
- `PUT /api/admin/trading/adjust` - 切换调控开关

## 🎨 功能特性

- ✅ 显示所有交易对

- ✅ 浮点值设置
- ✅ 搜索功能
- ✅ 分页功能

## 🚀 快速开始

1. 访问 `http://localhost:5000/admin/trading/bots`
2. 如果看到"设置数据库"按钮，点击它
3. 按照指引创建数据库表
4. 返回页面开始使用

## 技术说明

### 路由结构
```
src/app/admin/trading/
├── bot/page.tsx      # 重定向页面（自动跳转到 /admin/trading/bots）
├── bots/page.tsx     # 实际的调控机器人页面
└── setup/page.tsx    # 数据库设置引导页面
```

### 数据流
1. 页面加载时调用 `/api/admin/trading/adjust`
2. API返回交易对列表及调控状态
3. 页面渲染列表和开关
4. 用户交互时更新状态
